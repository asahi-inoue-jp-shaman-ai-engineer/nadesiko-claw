# MEMORY.md - ヤマトの記憶ルール

## 記憶の種類

| 種類 | 保存先 | 例 |
|---|---|---|
| 会話から学んだこと | `yamato_memories` | あさひの好み・癖・ビジョン |
| ペルソナ・自己認識 | `yamato_persona` | Soul.md, ASAHI_MD |
| 自律学習ログ | `yamato_learnings` | なでしこ文法、日本語NLP |
| 会話セッション | `yamato_sessions` | 開始/終了/処理状態 |

## [MEMORY]タグルール

AIレスポンスにこのタグを含めると自動でSupabaseに保存される：

```
[MEMORY category="insight" importance="4"]
あさひは霊性と技術を統合するビジョンを持っている。
イボガ・シャーマンとしての感性が開発の判断に影響する。
[/MEMORY]
```

### categoryの値

| 値 | 意味 |
|---|---|
| `insight` | あさひについての気づき |
| `asahi_trait` | あさひの特性・好み・癖 |
| `technical` | 技術的な発見・解決策 |
| `nadesiko` | なでしこ3の知見 |
| `kotodama` | 言霊・日本語霊読法の理解 |
| `general` | その他 |

### importanceの値

| 値 | 基準 |
|---|---|
| 5 | 絶対忘れてはいけない核心 |
| 4 | 重要。定期的に参照したい |
| 3 | 標準。文脈があれば使う |
| 2 | あってもなくてもいい |
| 1 | 参考程度 |

## 何を記憶しないか

- 一時的な確認事項（「今日は？」など）
- エラーの詳細ログ（logsディレクトリへ）
- 技術的な手順（skills/へ）
- 仕様の採用理由（DECISIONS.mdへ）

## クロンジョブのスケジュール

| 頻度 | 処理 |
|---|---|
| 毎日 | 前日の会話から記憶を抽出・整理 |
| 毎週 | importance上位記憶でSoul.mdを更新 |
| 毎月 | 低importance記憶をアーカイブ |
