# NADESIKO.md - なでしこ3 実装ガイド

## 実行環境

| 環境 | コマンド | 用途 |
|---|---|---|
| cnako3 | `cnako3 file.nako3` | サーバー・バッチ・フック |
| wnako3 | `<script type="なでしこ3">` | ブラウザUI |

## 文字列リテラル（重要）

| 記法 | 変数展開 | 使い所 |
|---|---|---|
| `「文字列」` | あり `{変数}` | 通常の文字列 |
| `『文字列』` | **なし** | JSON・正規表現 |
| `🌿...🌿` | **なし** | JS実行の長いブロック |
| `🌴...🌴` | あり | JS実行内で変数を使う |

**NE-001**: `「{"key":"val"}」` はエラー → 必ず `『{"key":"val"}』` を使う

## よく使う構文

```nako3
# 条件
もし 条件 ならば
    処理
ここまで

# ループ
配列を反復
    対象を表示。
ここまで

# エラー処理
エラー監視
    危険な処理。
エラーならば
    「失敗: {エラーメッセージ}」と表示。
ここまで

# 関数（カタカナ名推奨・命令名と衝突しない）
●（入力を）ヤマト処理とは
    それは 入力 & 「変換済み」
ここまで
```

## 落とし穴クイックリスト

| # | 症状 | 解決 |
|---|---|---|
| NE-001 | `「{...}」`でJSONエラー | `『』`を使う |
| NE-003 | `もし`内で関数結果を直接比較できない | 変数に入れてから比較 |
| NE-004 | `でなければ`が不安定 | `= 偽 ならば`を使う |
| NE-014 | 関数内で`もし`後に`戻る`忘れ | `それはX。戻る。`で必ず返す |
| NE-016 | 変数名に`秒`が含まれると`秒待機`と連結 | `(変数)秒待機`と括弧で区切る |
| NE-019 | 関数名末尾に`送信`/`取得`/`話`等 → 命令と衝突 | カタカナ造語を使う（例: `AIコール`） |

## cnako3でのHTTP呼び出し

```nako3
# fetchは非同期のためJS実行では待てない
# → api_call.mjsをexecSync経由で呼ぶ（起動待機）
応答文字列 = 「node ./api_call.mjs」を起動待機
応答 = 応答文字列をJSONデコード
```
